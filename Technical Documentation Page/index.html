SQL

SQL (Structured Query Language) è un linguaggio di programmazione standard utilizzato per gestire e manipolare i database relazionali. È uno strumento essenziale per gli sviluppatori web poiché consente di interagire con i dati archiviati nei database, eseguendo operazioni come lettura, scrittura, aggiornamento ed eliminazione di dati.

  • SQL è un linguaggio dichiarativo, il che significa che gli sviluppatori specificano cosa vogliono ottenere senza dover descrivere come ottenerlo.
  • SQL è un linguaggio basato su set, il che significa che le query SQL operano su insiemi di dati piuttosto che su record singoli.
  • SQL è un linguaggio case-insensitive, il che significa che non fa distinzione tra maiuscole e minuscole.
Esempio di query SQL:
SELECT Colonna1, Colonna2 FROM Tabella WHERE Colonna1 = Valore;
DDL

Il Linguaggio di Definizione dei Dati (DDL) è un sottoinsieme di SQL utilizzato per definire la struttura dei database, inclusa la creazione, la modifica e l'eliminazione di tabelle, indici, vincoli e altri oggetti del database.

Creare una nuova tabella:

L'istruzione CREATE TABLE viene utilizzata per creare una nuova tabella nel database. Ecco la sintassi generale:


CREATE TABLE NomeTabella ( Colonna1 TipoDato1, Colonna2 TipoDato2, ... );
DML

Il Linguaggio di Manipolazione dei Dati (DML) è un sottoinsieme di SQL utilizzato per gestire i dati all'interno delle tabelle del database. Le istruzioni DML consentono di inserire, selezionare, aggiornare ed eliminare dati dalle tabelle.

Inserire nuovi dati:

L'istruzione INSERT INTO viene utilizzata per inserire nuovi record in una tabella. Ecco la sintassi generale:


INSERT INTO NomeTabella (Colonna1, Colonna2, ...) VALUES (Valore1, Valore2, ...);
Operatori

Gli operatori e le funzioni SQL sono utilizzati per eseguire operazioni su dati all'interno delle query. Gli operatori vengono utilizzati per confrontare valori, eseguire operazioni matematiche e logiche, mentre le funzioni vengono utilizzate per eseguire calcoli, manipolare stringhe e date, e altro ancora.


SELECT Colonna1, Colonna2 FROM Tabella WHERE Colonna1 = Valore AND Colonna2 > 10;
Join

Un JOIN è un'operazione che combina le righe di due o più tabelle in base a una relazione tra di esse. Ci sono diversi tipi di JOIN disponibili in SQL, tra cui INNER JOIN, LEFT JOIN, RIGHT JOIN e FULL JOIN.

  • INNER JOIN: restituisce solo le righe che hanno una corrispondenza nelle tabelle collegate.
  • LEFT JOIN: restituisce tutte le righe dalla tabella sinistra e le righe corrispondenti dalla tabella destra.
  • RIGHT JOIN: restituisce tutte le righe dalla tabella destra e le righe corrispondenti dalla tabella sinistra.
SELECT Colonne FROM Tabella1 JOIN Tabella2 ON Tabella1.Colonna = Tabella2.Colonna;
Sottoqueries

Una sottoquery è una query annidata all'interno di un'altra query. Le sottoquery possono essere utilizzate per eseguire operazioni complesse, come filtrare i risultati di una query principale o ottenere valori aggregati da più tabelle.


SELECT Colonne FROM Tabella WHERE Colonna IN (SELECT Colonna FROM AltraTabella);
Viste

Una vista è una tabella virtuale che rappresenta una selezione di dati da una o più tabelle. Le viste possono essere utilizzate per semplificare le query complesse, nascondere i dettagli della struttura del database e garantire la sicurezza dei dati.


CREATE VIEW NomeVista AS SELECT Colonne FROM Tabella WHERE Condizione;
Stored Procedures

I stored procedure e le funzioni sono blocchi di codice SQL che possono essere riutilizzati in più query o applicazioni. Le stored procedure sono salvate nel database e possono accettare parametri di input e restituire valori di output, mentre le funzioni restituiscono un singolo valore.


CREATE PROCEDURE NomeProcedura AS BEGIN -- Codice della procedura qui END;
Transazioni

Una transazione è un insieme di operazioni che devono essere eseguite in modo atomico, cioè o tutte le operazioni vengono completate con successo o nessuna di esse viene eseguita. Le transazioni sono utilizzate per garantire l'integrità dei dati e la coerenza del database.


BEGIN TRANSACTION; -- Operazioni di aggiornamento qui COMMIT TRANSACTION;
